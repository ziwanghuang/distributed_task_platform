## 1. 补偿器架构总览

```
┌──────────────────────────────────────────────────────────────┐
│                    4 大补偿器                                  │
│                                                               │
│  RetryCompensator          RescheduleCompensator              │
│  ┌────────────────────┐    ┌────────────────────┐             │
│  │ 状态: FAILED_       │    │ 状态: FAILED_       │             │
│  │      RETRYABLE      │    │      RESCHEDULED    │             │
│  │ 条件: next_retry    │    │ 条件: 非分片父任务    │             │
│  │      <= now         │    │                     │             │
│  │ 动作: Runner.Retry  │    │ 动作: Runner.        │             │
│  └────────────────────┘    │      Reschedule     │             │
│                            └────────────────────┘             │
│  InterruptCompensator      ShardingCompensator                │
│  ┌────────────────────┐    ┌────────────────────┐             │
│  │ 状态: RUNNING       │    │ 父任务 sharding_    │             │
│  │ 条件: deadline      │    │ parent_id = 0      │             │
│  │      <= now         │    │ 状态: RUNNING        │             │
│  │ 动作: gRPC          │    │ 动作: 聚合子任务     │             │
│  │      Interrupt      │    │      更新父任务      │             │
│  └────────────────────┘    └────────────────────┘             │
└──────────────────────────────────────────────────────────────┘
```

### 通用模式

所有补偿器遵循相同的循环模式：

```
for {
    startTime := now()
    items := FindXxx(ctx, batchSize)      // 查找需要补偿的记录
    for _, item := range items {
        process(ctx, item)                 // 逐个处理
    }
    elapsed := since(startTime)
    if elapsed < minDuration {
        sleep(minDuration - elapsed)       // 防空转保护
    }
}
```

## 2. RetryCompensator（重试补偿）

**包路径**：`internal/compensator/retry.go` / `retryv2.go`

### 触发条件

- `status = FAILED_RETRYABLE`
- `next_retry_time <= now()`
- `retry_count < maxRetryCount`

### 处理流程

```
1. FindRetryableExecutions(batchSize)
2. 遍历每条记录:
   Runner.Retry(ctx, execution)
     └── 重新抢占任务 → 排除上次失败节点 → 异步发起远程调用
```

### 重试策略计算

由 `ExecutionService.updateRetryState` 处理：

```
retryStrategy := NewRetry(task.RetryConfig)  // 指数退避 or 固定间隔
duration, shouldRetry := retryStrategy.NextWithRetries(retryCount + 1)
if shouldRetry:
    execution.NextRetryTime = now + duration
    execution.RetryCount++
    → 状态保持 FAILED_RETRYABLE
else:
    → 状态强制改为 FAILED（终态）
    → 发送完成事件
```

## 3. RescheduleCompensator（重调度补偿）

**包路径**：`internal/compensator/reschedule.go` / `reschedulev2.go`

### 触发条件

- `status = FAILED_RESCHEDULED`
- 非分片父任务

### 处理流程

```
1. FindReschedulableExecutions(batchSize)
2. 遍历每条记录:
   Runner.Reschedule(ctx, execution)
     └── 重新抢占 → 指定到原节点 → 异步发起远程调用
```

**重调度 vs 重试**：重试排除故障节点，重调度指定到原节点（适合需保持状态的场景）。

## 4. InterruptCompensator（超时中断补偿）

**包路径**：`internal/compensator/interrupt.go` / `interruptv2.go`

### 触发条件

- `status = RUNNING`
- `deadline <= now()`

### 处理流程

```
1. FindTimeoutExecutions(batchSize)
2. 遍历每条记录:
   a. 通过 gRPC 发送 Interrupt 请求到 Executor
   b. 成功 → UpdateState(FAILED)
   c. 失败 → 日志记录，下次循环重试
```

## 5. ShardingCompensator（分片聚合补偿）

**包路径**：`internal/compensator/sharding.go` / `shardingv2.go`

### 触发条件

- `sharding_parent_id = 0`（即分片父任务）
- `status = RUNNING`

### 处理流程

```
1. FindShardingParents(offset, batchSize)
2. 遍历每个父任务:
   a. FindShardingChildren(parentID) → 获取所有子任务
   b. 聚合状态:
      - 全部 SUCCESS → 父任务 = SUCCESS
      - 有一个 FAILED → 父任务 = FAILED
      - 有未完成的 → 跳过，等下次循环
   c. 更新父任务状态 + 释放 Task 抢占锁 + 更新下次执行时间
```

## 6. V1 vs V2 差异

| 维度 | V1 | V2 |
|------|-----|-----|
| 启动方式 | `for` 循环 + `select` | `ShardingLoopJob.Run(ctx)` |
| 数据扫描 | 全表 | 按分片（context 携带 Dst） |
| 并发控制 | 无（多节点扫同一表） | 分布式锁 + 资源信号量 |
| 防空转 | `minDuration` sleep | ShardingLoopJob 内置 |
| 适用场景 | 单机 / 少量节点 | 多节点大规模 |

### V2 启动示例

```go
func (r *RetryCompensatorV2) Start(ctx context.Context) {
    loopjob.NewShardingLoopJob(r.dlockClient, "retryKey", r.retry, r.executionStr, r.sem).Run(ctx)
}
```

## 7. 配置参数

```yaml
compensator:
  retry:
    maxRetryCount: 3            # 最大重试次数
    prepareTimeoutWindowMs: 10s # PREPARE 状态超时窗口
    batchSize: 100              # 批量处理数
    minDuration: 1s             # 最小循环间隔（防空转）
  reschedule:
    batchSize: 100
    minDuration: 1s
  sharding:
    batchSize: 100
    minDuration: 1s
  interrupt:
    batchSize: 100
    minDuration: 1s
```
