## 1. Task（任务定义）

**包路径**：`internal/domain/task.go`

Task 是调度的最小单元——定义"什么时候执行、怎么执行、在哪执行"。

### 1.1 结构体字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `ID` | `int64` | 任务唯一标识（Snowflake 生成） |
| `Name` | `string` | 任务名称（DAG 中用于节点匹配） |
| `CronExpr` | `string` | Cron 表达式，支持 6 位（秒级精度） |
| `ExecExpr` | `string` | DAG DSL 表达式（仅 Plan 类型任务使用） |
| `Type` | `TaskType` | 任务类型：`normal` / `plan` |
| `ExecutionMethod` | `TaskExecutionMethod` | 执行方式：`LOCAL` / `REMOTE` |
| `SchedulingStrategy` | `SchedulingStrategy` | 调度策略：`CPU_PRIORITY` / `MEMORY_PRIORITY` |
| `GrpcConfig` | `*GrpcConfig` | gRPC 远程调用配置 |
| `HTTPConfig` | `*HTTPConfig` | HTTP 远程调用配置 |
| `RetryConfig` | `*RetryConfig` | 重试策略配置 |
| `MaxExecutionSeconds` | `int64` | 最大执行秒数（默认 24 小时），用于 Deadline 计算 |
| `ScheduleNodeID` | `string` | 当前持有该任务的调度节点 UUID |
| `ScheduleParams` | `map[string]string` | 调度参数（分片偏移量、处理进度等） |
| `ShardingRule` | `*ShardingRule` | 分片规则（为 nil 则为普通任务） |
| `NextTime` | `int64` | 下次调度时间（毫秒时间戳） |
| `Status` | `TaskStatus` | 任务状态 |
| `Version` | `int64` | 乐观锁版本号 |
| `PlanID` | `int64` | 所属 Plan 的 ID（0 = 独立任务） |
| `PlanExecID` | `int64` | 所属 Plan 执行记录 ID |
| `CTime` | `int64` | 创建时间戳（毫秒） |
| `UTime` | `int64` | 更新时间戳（毫秒） |

### 1.2 TaskStatus 状态机

```
              创建                      调度器抢占
  INACTIVE ─────────► ACTIVE ──────────────────► PREEMPTED
                        ▲                            │
                        │      执行完毕/释放            │
                        └────────────────────────────┘
```

| 状态 | 值 | 含义 |
|------|-----|------|
| `ACTIVE` | `"ACTIVE"` | 可被调度 |
| `PREEMPTED` | `"PREEMPTED"` | 已被某个 Scheduler 节点抢占 |
| `INACTIVE` | `"INACTIVE"` | 已停用，不参与调度 |

### 1.3 子结构体

#### GrpcConfig

```go
type GrpcConfig struct {
    ServiceName string            `json:"serviceName"` // etcd 中注册的服务名
    Params      map[string]string `json:"params"`      // 业务自定义参数
}
```

#### HTTPConfig

```go
type HTTPConfig struct {
    Endpoint string            `json:"endpoint"` // HTTP 调用端点
    Params   map[string]string `json:"params"`   // 业务自定义参数
}
```

#### RetryConfig

```go
type RetryConfig struct {
    MaxRetries      int32 `json:"maxRetries"`      // 最大重试次数
    InitialInterval int64 `json:"initialInterval"` // 初始重试间隔（毫秒）
    MaxInterval     int64 `json:"maxInterval"`     // 最大重试间隔（毫秒）
}
```

- `ToRetryComponentConfig()` → 转换为 `retry.Config`，类型为 `"exponential"` 指数退避策略

#### ShardingRule

```go
type ShardingRule struct {
    Type                  ShardingRuleType           `json:"type"`   // range / weighted-dynamic-range
    Params                map[string]string          `json:"params"` // 策略参数
    ExecutorNodeInstances []registry.ServiceInstance  `json:"-"`     // 运行时注入的执行节点列表
}
```

- `ToScheduleParams() []map[string]string` → 根据分片类型计算每个子任务的调度参数
  - **Range**：按 `step` × `totalNums` 固定步长切分，每片 `{start, end}`
  - **WeightedDynamicRange**：按节点权重比例切分 `total_tasks`，能力强的节点分更多数据

### 1.4 核心方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `CalculateNextTime` | `() (time.Time, error)` | 根据 CronExpr 计算下次执行时间 |
| `UpdateScheduleParams` | `(params map[string]string)` | 智能合并调度参数（nil=不操作，空map=清空，有值=覆盖合并） |

## 2. TaskExecution（执行记录）

**包路径**：`internal/domain/task_execution.go`

每次调度产生一条 TaskExecution，记录完整的执行快照（反范式冗余 Task 全部字段）。

### 2.1 结构体字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `ID` | `int64` | 执行记录唯一标识 |
| `ShardingParentID` | `*int64` | 分片语义指针：`nil`=普通任务，`0`=分片父，`>0`=分片子（值为父 ID） |
| `Deadline` | `int64` | 执行截止时间（毫秒时间戳），超时触发 InterruptCompensator |
| `ExecutorNodeID` | `string` | 实际执行节点 ID |
| `StartTime` | `int64` | 开始时间（毫秒时间戳） |
| `EndTime` | `int64` | 结束时间（毫秒时间戳） |
| `RetryCount` | `int64` | 已重试次数 |
| `NextRetryTime` | `int64` | 下次重试时间（毫秒时间戳） |
| `RunningProgress` | `int32` | 执行进度 0-100 |
| `Status` | `TaskExecutionStatus` | 执行状态 |
| `CTime` | `int64` | 创建时间戳 |
| `UTime` | `int64` | 更新时间戳 |
| `Task` | `Task` | 创建时刻从 Task 冗余的完整信息 |

### 2.2 TaskExecutionStatus 状态机

```
                    ┌─────────────────────────────────────┐
                    │                                     │
                    ▼                                     │
  PREPARE ──► RUNNING ──► SUCCESS                         │
                  │                                       │
                  ├──► FAILED (终态)                       │
                  │                                       │
                  ├──► FAILED_RETRYABLE ─── 重试 ──► RUNNING
                  │
                  └──► FAILED_RESCHEDULED ─ 重调度 ─► (新 Execution)
```

| 状态 | 值 | 终态？ | 含义 |
|------|----|-------|------|
| `UNKNOWN` | `"UNKNOWN"` | — | 未知（默认/异常） |
| `PREPARE` | `"PREPARE"` | 否 | 已创建执行记录，等待执行节点响应 |
| `RUNNING` | `"RUNNING"` | 否 | 正在执行中 |
| `SUCCESS` | `"SUCCESS"` | ✅ | 执行成功 |
| `FAILED` | `"FAILED"` | ✅ | 执行失败（不可恢复） |
| `FAILED_RETRYABLE` | `"FAILED_RETRYABLE"` | 否 | 执行失败（可重试，等待 RetryCompensator） |
| `FAILED_RESCHEDULED` | `"FAILED_RESCHEDULED"` | 否 | 执行失败（需重调度，等待 RescheduleCompensator） |

### 2.3 核心方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `MergeTaskScheduleParams` | `(params map[string]string)` | 合并重调度参数到执行记录 |
| `GRPCParams` | `() map[string]string` | 合并业务参数 + 调度参数 + 超时参数，供 gRPC 调用使用 |
| `HTTPParams` | `() map[string]string` | 同上，供 HTTP 调用使用 |
| `IsTerminalStatus` | `() bool` | 判断是否为终态（Success 或 Failed） |

> **参数优先级**：调度参数 > 业务参数（同名 key 被覆盖）

## 3. Plan（DAG 工作流）

**包路径**：`internal/domain/plan.go`

Plan 由多个 Task 通过 DSL 表达式定义依赖关系，运行时构建为 DAG 图。

### 3.1 结构体字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `ID` | `int64` | Plan ID |
| `Name` | `string` | Plan 名称 |
| `CronExpr` | `string` | 调度 Cron 表达式 |
| `ExecExpr` | `string` | DSL 依赖表达式（如 `"A -> (B && C) -> D"` ） |
| `Execution` | `TaskExecution` | Plan 整体执行状态 |
| `ExecPlan` | `parser.TaskPlan` | 解析后的 AST（DAG 图结构接口） |
| `ScheduleNodeID` | `string` | 当前持有节点 ID |
| `Steps` | `[]*PlanTask` | 所有节点任务（平铺） |
| `Root` | `[]*PlanTask` | 根节点（DAG 入口，无前驱） |
| `ScheduleParams` | `map[string]string` | 调度参数 |
| `NextTime` | `int64` | 下次执行时间 |
| `Status` | `TaskStatus` | Plan 状态 |
| `Version` | `int64` | 乐观锁版本号 |

### 3.2 方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `RootTask` | `() []*PlanTask` | 返回 DAG 根节点列表 |
| `GetTask` | `(name string) (*PlanTask, bool)` | 按名称查找节点 |

## 4. PlanTask（工作流节点）

**包路径**：`internal/domain/plantask.go`

DAG 图中的每个节点，包含前驱/后继关系和 AST 节点信息。

### 4.1 结构体字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `PreTask` | `[]*PlanTask` | 前驱节点列表 |
| `TaskExecution` | `TaskExecution` | 该节点当前的执行状态 |
| `AstPlanNode` | `parser.PlanNode` | AST 解析的邻接信息 |
| `NextTask` | `[]*PlanTask` | 后继节点列表（运行时动态计算） |
| (嵌入) | `Task` | 任务定义 |

### 4.2 核心方法

| 方法 | 签名 | 说明 |
|------|------|------|
| `SetPre` | `(tasks map[string]*PlanTask) error` | 根据 AST 设置前驱关系 |
| `SetNext` | `(tasks map[string]*PlanTask) error` | 根据 AST + 执行状态设置后继关系 |
| `NextStep` | `() []*PlanTask` | 返回后继节点列表 |
| `CheckPre` | `() bool` | 检查前驱是否满足：AND=全成功，OR/Condition=任一成功 |

## 5. Report / ExecutionState（状态上报）

**包路径**：`internal/domain/report.go`

执行节点通过 gRPC → Kafka 上报执行状态。

### Report

```go
type Report struct {
    ExecutionState ExecutionState `json:"executionState"`
}
```

### ExecutionState

| 字段 | 类型 | 说明 |
|------|------|------|
| `ID` | `int64` | 执行记录 ID |
| `TaskID` | `int64` | 任务 ID |
| `TaskName` | `string` | 任务名称 |
| `Status` | `TaskExecutionStatus` | 执行状态 |
| `RunningProgress` | `int32` | 进度 0-100 |
| `RequestReschedule` | `bool` | 执行节点是否主动请求重调度 |
| `RescheduleParams` | `map[string]string` | 重调度参数（offset、limit 等） |
| `ExecutorNodeID` | `string` | 执行节点标识 |

### BatchReport

```go
type BatchReport struct {
    Reports []*Report `json:"reports"`
}
```

## 6. 错误码一览

**包路径**：`internal/errs/error.go`

| 错误 | 含义 | 触发场景 |
|------|------|---------|
| `ErrTaskPreemptFailed` | 任务抢占失败 | CAS 版本号不匹配 |
| `ErrTaskRenewFailed` | 任务续约失败 | 任务已被其他节点接管 |
| `ErrTaskReleaseFailed` | 任务释放失败 | CAS 校验失败 |
| `ErrTaskUpdateNextTimeFailed` | 更新下次执行时间失败 | 乐观锁冲突 |
| `ErrExecutionNotFound` | 执行记录不存在 | ID 查询无结果 |
| `ErrInvalidTaskExecutionStatus` | 状态非法 | 从终态再次迁移 |
| `ErrInterruptTaskExecutionFailed` | 中断执行失败 | Executor 返回失败 |
| `ErrInvalidTaskCronExpr` | 无效 Cron 表达式 | 解析失败 |
| `ErrInvalidTaskShardingRule` | 分片规则非法 | JSON 格式错误 |
| `ErrTaskShardingRuleNotFound` | 分片规则缺失 | 分片任务未配置规则 |
| `ErrSetExecutionStateRunningFailed` | 设置运行状态失败 | DB 更新失败 |
| `ErrUpdateExecutionStatusFailed` | 更新状态失败 | 通用状态更新错误 |
| `ErrExecutionMaxRetriesExceeded` | 超过最大重试次数 | 重试上限已达 |
| `ErrExecutionStateHandlerNotFound` | 状态处理器未找到 | 非法状态值 |
| `ErrInitPlanFailed` | Plan 初始化失败 | DSL 任务名与 DB 不匹配 |
| `ErrExceedLimit` | 资源超出限制 | 信号量已满 |

## 7. 类型常量速查表

### TaskType

| 值 | 说明 |
|----|------|
| `normal` | 普通任务（含分片） |
| `plan` | DAG 工作流 |

### TaskExecutionMethod

| 值 | 说明 |
|----|------|
| `LOCAL` | 本地函数调用（测试用） |
| `REMOTE` | 远程 gRPC/HTTP 调用 |

### SchedulingStrategy

| 值 | 说明 |
|----|------|
| `CPU_PRIORITY` | CPU 空闲率优先 |
| `MEMORY_PRIORITY` | 可用内存优先 |

### ShardingRuleType

| 值 | 说明 |
|----|------|
| `range` | 固定步长范围分片 |
| `weighted-dynamic-range` | 按节点权重动态分片 |

### Protobuf ExecutionStatus

| 枚举值 | 编号 | 映射到 |
|--------|------|--------|
| `UNKNOWN` | 0 | `TaskExecutionStatusUnknown` |
| `RUNNING` | 1 | `TaskExecutionStatusRunning` |
| `FAILED_RETRYABLE` | 2 | `TaskExecutionStatusFailedRetryable` |
| `FAILED_RESCHEDULABLE` | 3 | `TaskExecutionStatusFailedRescheduled` |
| `FAILED` | 4 | `TaskExecutionStatusFailed` |
| `SUCCESS` | 5 | `TaskExecutionStatusSuccess` |
